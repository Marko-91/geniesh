# auto-improve — Closed-loop self-improvement for geniesh

## Core idea

Auto-improve is a self-healing loop that makes geniesh better at understanding code by continuously testing itself.

```
  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌────────┐
  │   Eval   │───▶│ Analyze  │───▶│   Fix    │───▶│ Retest │──┐
  │ benchmark│    │ with LLM │    │  source  │    │  eval  │  │
  └──────────┘    └──────────┘    └──────────┘    └────────┘  │
      ▲                                                       │
      └───────────────────────────────────────────────────────┘
```

It tests geniesh against a set of benchmark repos (Express, Flask, Gin, Ripgrep, Zod), each with ~12–15 questions about the codebase. For each question, geniesh must retrieve the correct files and symbols. The system:

1. **Eval** — runs geniesh against each benchmark question, measures file recall and symbol recall
2. **Analyze** — feeds the results to an LLM that knows the geniesh architecture (BFS traversal, relation graph, symbol extraction, grep bridge, RAG)
3. **Fix** — the LLM suggests specific code changes to the source files
4. **Retest** — re-runs the eval to measure if scores improved
5. **Loop** — repeats until no further improvement or max iterations reached
6. **PR** — (optional) pushes improvements as a GitHub branch + PR

## Why it works

The system exploits the fact that geniesh's retrieval quality depends on a few concentrated bottlenecks:

- **symbol-utils.js** — regex patterns that define what counts as a "symbol" in each language
- **relations.js** — how imports/exports build the relation graph
- **context-builder.js** — BFS parameters (frontier size, grep bridge, chunking)
- **grep.js** — file content caching and context windows

The LLM can spot patterns across all 5 repos at once — a symbol missing in both Go and Rust suggests the regex is too narrow; a file missed in both Flask and Express suggests the BFS frontier is too small.

## Usage

```bash
geniesh self-improve              # 5 iterations
geniesh self-improve 10           # 10 iterations
geniesh self-improve --model qwen3.4:7b  # use a different model
```

## Prerequisites

- `.reports/auto-*-benchmark.json` files (generated by `geniesh benchmark generate`)
- Ollama running with an LLM model
- Git configured for commit/PR support (optional)
